# 마비노기 모바
일 파밍 자동화 (MabinogiAuto)

> 어비스 / 던전 / 사냥터 3개 콘텐츠 지원

게임 화면을 OCR(글자 인식)로 읽어 어비스/던전/사냥터를 자동으로 **입장 → 클리어 대기 → 보상 수령 → 재입장 반복**하는 자동화 도구입니다.

---

## 빠른 시작

1. 마비노기 모바일 실행 (**창 모드** 권장)
2. 콘텐츠에 맞는 시작 화면으로 이동:

   | 콘텐츠 | 시작 화면 |
   |---|---|
   | 어비스 | 어비스 던전 선택 화면 |
   | 던전 | 원하는 던전의 구역 선택 화면 |
   | 사냥터 | 원하는 사냥터의 첫 화면 |

3. `MabinogiAuto.exe` 더블클릭 → 관리자 권한(UAC) "예"
4. 컨트롤 패널에서 콘텐츠 선택 + 상세 설정 + 반복 방식(무한/횟수/시간) 지정
5. **[시작]** 클릭

### 버튼 / 단축키

| 버튼 | 동작 |
|---|---|
| 시작 (F9) | 반복 시작 |
| 안전 중지 (F9) | 진행 중인 회차를 끝까지 마치고 멈춤 (다시 누르면 예약 취소) |
| 즉시 중지 (F10) | 바로 종료 |
| 권장 창 모드 | 게임 창을 OCR 최적 크기(1272 또는 1908)로 즉시 변경 |
| 설정 저장 | 설정만 저장 (다음 회차부터 적용) |

> F9/F10은 게임 화면에 포커스가 있어도 동작합니다.

## 파일 구성

| 파일 | 설명 |
|---|---|
| `MabinogiAuto.exe` | **시작점.** 더블클릭으로 실행. 단일 파일이라 다른 PC엔 이것만 복사하면 됨 |
| `mabinogi_gui.ps1` | 컨트롤 패널 원본 (exe에 내장됨) |
| `mabinogi_run_once.ps1` | 1회차 실행 본체 — 클릭·OCR·키 입력 (exe에 내장됨) |
| `config.json` | 설정 파일 (좌표·대기시간·키 등, 한국어 설명 포함) |
| `rdp_redirect_console.ps1` | RDP 자동 전환 내부 스크립트 |

- exe는 실행 시 스크립트를 `%LOCALAPPDATA%\MabinogiAuto`로 추출해 실행합니다.
- exe 업데이트 시 좌표/구조는 자동 최신화되고 사용자 설정은 유지됩니다.

## 콘텐츠별 동작 흐름

### 어비스 (허상의 정박지 / 광기의 동굴 / 흩어진 물길)

1. 던전 카드 클릭 → 상세 화면 확인 → 혼자하기/함께하기 탭 → 난이도 클릭
2. 입장 — 함께하기는 매칭 방식 4종 지원:

   | 매칭 | 동작 |
   |---|---|
   | 우연한 만남 | 토글 켜고 입장하기 → 인원이 모이면 자동 입장 |
   | 파티 찾기 | 파티 찾기 → 매칭 완료 시 자동 입장 |
   | 파티(파티장) | 직접 짠 파티 그대로 입장하기 → 전원 준비되면 자동 입장 |
   | 파티(파티원) | 필드 대기 중 파티장이 입장 시작하면 '준비 완료' 자동 클릭 |

3. 입장 감지 → 자동출발/음식 키 → 클리어 대기 → 화면 터치 → 나가기 → 반복

### 던전

- 선택/옵션/결과 화면 자동 인식 → 난이도 + 스테이지 설정
- 은동전 소탕/더블 루팅 카드 설정 (소진 시 폴백 옵션)
- 클리어 대기 → 엔딩 컷신 넘김 → **다시 하기** → 반복

### 사냥터

- 난이도 선택 → 은동전(사냥 임무)/더블 루팅 카드 설정
- 파티찾기 또는 바로 입장 → 사냥 완료 대기 → **새 임무 선택** → 반복
- 은동전 10개 미만이면 자동 종료

## 내장된 안전장치

- **자동 부활**: 행동불능 감지 → 남은 횟수 있으면 제자리 부활, 없으면 여신상 부활
- **자동사냥 꺼짐 감시**: 부활/이동 후 자동사냥이 꺼져 있으면 Space 재입력
- **보스/엔딩 컷신** '장면 넘기기' 자동 클릭
- **아침 6시 리셋 체인** 자동 처리: 출석부 → 스텔라 픽 → 공지/이벤트 팝업
- **전투가 긴 판 보호**: 클리어 대기 한도를 넘겨도 전투 진행 중이면 자동 연장 (최대 설정값의 3배)
- **화면 캡처 실패 시 일시정지**: RDP 최소화/끊김 시 자동 감지 후 복구되면 이어서 진행
- **자동 전면화**: 게임 창이 가려져 감지 실패 시 자동으로 앞에 가져옴
- **오류 시 진단 덤프**: 실패 순간 스크린샷 + 로그 세트(`error_시각.png` + `error_시각.log`) 저장

## RDP(원격 데스크톱) 사용 시

| 상황 | 동작 |
|---|---|
| RDP 창 열어둠 | 정상 작동 |
| RDP 최소화 | 자동화 일시정지 (다시 열면 이어감) |
| RDP 닫기/끊기 | 자동으로 본체 화면 전환 → 계속 작동 |

> **RDP 최소화 대신 창을 닫을 것** — 최소화 상태에서는 화면 캡처가 되지 않아 일시정지됩니다.

## 설정 변경 (config.json)

대부분의 설정은 GUI에서 바꾸고 [설정 저장]을 누르면 됩니다.
세부 설정은 `config.json`을 메모장으로 열어 수정하면 다음 회차부터 적용됩니다.

| 항목 | 설명 |
|---|---|
| `timeoutsSeconds.dungeonClear` | 클리어 대기 시간 (기본 600초, 전투 중이면 자동 연장) |
| `timeoutsSeconds.partyMatching` | 파티 매칭 대기 (기본 300초) |
| `afterEntry.keys` | 입장 후 누를 키 |
| `rdp.autoConsoleRedirect` | RDP 끊김 시 본체 전환 (`false` = 끄기) |
| `diagnostics.keepScreenshots` | 오류 스크린샷 보관 개수 (기본 20) |

`config.json`을 삭제하면 내장 기본값으로 실행됩니다.

## 문제가 생기면

1. `Log\mabinogi_run_once.log` 마지막 줄 확인
   - `화면 캡처 실패 - RDP 창이 최소화` → RDP 창을 다시 열기
   - `화면 캡처 실패 - RDP 연결이 끊긴 상태` → 몇 초 내 자동 복구 대기
   - `[오류]` + `[진단]` → 같이 저장된 `error_*.png` 확인
2. 흔한 원인
   - 게임이 전체화면으로 바뀜 → 창 모드로 변경
   - 시작 화면 위치가 다름 → 각 콘텐츠의 권장 시작 화면에서 재시작
   - '다른 자동화 인스턴스가 이미 실행 중' → 작업 관리자에서 `powershell.exe` 종료 후 재시작
