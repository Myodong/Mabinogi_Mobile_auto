## 3. 개선 아이디어 (낮은 우선순위 기술 부채)

> 지금 동작에는 문제 없음. 각 항목의 **진행 시점**이 되면 착수.
> 완료된 항목(대기 함수 통합 / 출석 헬퍼 추출 / OCR 임시 PNG 제거 / 던전·사냥터 공통 블록 추출)은 해결됨 섹션 참고.

### (1) 폴링당 화면 캡처 공유 (OCR 잔여 효율화)
- **목적**: 폴링 1회에 여러 감지가 각자 화면을 캡처(3~6회)하는 것을 1회 캡처 + 영역 잘라 쓰기로 축소.
- **위험**: 감지 사이에 화면이 바뀌는 타이밍 이슈 → 회귀 위험이 이득보다 큼.
- **진행 시점**: 폴링 속도가 실제로 부족하다고 확인될 때만.
- 참고: OCR 엔진 교체/메모리 리딩/템플릿 매칭은 검토 후 기각 (무의존 단일 exe·다중 PC·RDP 제약에서는 현 구조가 최선).

### (2) GUI 로그 표시를 바이트 오프셋 방식으로
- **목적**: 로그가 커질 때 매 갱신마다 파일 전체를 다시 읽는 비용 제거.
- 현재는 회차마다 로그 삭제 + 표시 20만 자 제한이라 비용이 작음.
- **진행 시 필수 처리**: 로그 삭제·재생성 시 오프셋 초기화 / 파일이 오프셋보다 작아지면 초기화 / 불완전한 마지막 줄 보관 / UTF-8 한글 바이트 경계 처리 (단순 Seek만 추가하면 글자 깨짐·줄 유실 발생).
- **진행 시점**: 실제 로그가 수백 KB 이상 커지는 것이 확인될 때.
